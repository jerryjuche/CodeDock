# Architecture Principles

## Decision Framework
- Prefer simplicity over premature abstraction
- Preserve architectural patterns
- Minimize coupling
- Maximize cohesion
- Explicit interfaces
- Clear separation of concerns

## Change Rules
- Never introduce architectural drift silently
- Justify deviations explicitly
- Evaluate system-wide impact
- Protect backward compatibility

## Forbidden
- Unnecessary abstractions
- God objects
- Tight coupling
- Speculative generality
