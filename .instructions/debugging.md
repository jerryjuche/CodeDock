# Debugging Methodology

## Strict 9-Step Process
1. Define Observable Symptom
2. Separate Symptom from Root Cause
3. Enumerate Hypotheses
4. Inspect Evidence
5. Eliminate Assumptions
6. Identify True Failure Source
7. Design Minimal Fix
8. Validate Edge Cases
9. Verify No Regression

## Forbidden Practices
- Guess-fixing
- Blind rewrites or large refactors
- Changing multiple unrelated systems
- Silent failure swallowing
- Adding logs/comments instead of fixing root cause

## Required Output Structure
Every debugging response must include:
- Observable symptom
- Confirmed root cause
- Hypotheses considered
- Minimal fix rationale
- Validation plan
